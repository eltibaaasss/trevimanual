x · Área de respeto

Área de Respeto

El logo siempre debe tener un espacio libre equivalente a la altura de la "x" del logotipo. Nunca ubicar otros elementos dentro de esta zona.

  • →Tamaño mínimo: 80px de ancho
  • →No distorsionar proporciones
  • →No aplicar sombras ni efectos
  • →No usar sobre fondos complejos

04 — Imágenes

Estilo Visual

Las imágenes de Trevi deben reflejar la vida real de Bogotá: auténtica, vibrante y diversa.

🎵
Música en vivo · Calidez, contraste
☕
Cafés y espacios · Luz natural
🎭
Arte y cultura · Colores cálidos
✦ Usar
  • →Fotografía con personas reales en Bogotá
  • →Imágenes con alto contraste y saturación moderada
  • →Ambientes nocturnos o al atardecer (energía urbana)
  • →Close-ups de detalles culturales únicos
✕ Evitar
  • →Stock photography genérico sin contexto colombiano
  • →Imágenes sobreexpuestas o con filtros extremos
  • →Fondos blancos de estudio (no refleja la marca)
  • →Composiciones demasiado perfectas o artificiales

07 — Reglas de Uso

Do's & Don'ts

Estas reglas no son sugerencias — son los límites del sistema para mantener la identidad coherente.

✦ Sí hacer
Usar Azul Trevi como color dominante en navegación y acciones primarias (60%).
Reservar Verde exclusivamente para estados "activo/abierto" y CTAs secundarios.
Usar el gradiente azul→verde en ilustraciones, fondos hero y piezas de marketing.
Usar Azul Niebla para fondos de tooltips y banners informativos en modo claro.
✕ No hacer
No mezclar Dorado y Coral — confunden el sistema semántico de la marca.
No usar Azul Claro como color primario sobre fondos blancos — pierde identidad.
No usar Coral para elementos positivos. Es exclusivo de errores y "cerrado".
No usar solo grises — siempre presencia de al menos un color de marca.

08 — Design Tokens

CSS & Variables

Los tokens son la fuente de verdad del sistema. Cópialos directamente en tu código para garantizar consistencia total.

trevi-tokens.css
/* ── TREVI DESIGN TOKENS ─────────────── */ /* Primarios */ --color-azul-deep: #1E2FBB; --color-azul: #3A4FE8; /* ← Principal */ --color-azul-light: #6E7FF5; --color-azul-mist: #EEF0FE; --color-verde-deep: #1A9E3A; --color-verde: #38D95B; /* ← Acento */ --color-verde-light: #7EEEA0; --color-verde-mist: #EDFBF1; /* Soporte */ --color-gold: #F5A623; /* Eventos */ --color-coral: #FF5C5C; /* Error · Cerrado */ /* Neutros */ --color-dark: #0B0C14; --color-dark-2: #13151F; --color-dark-3: #1E2030; --color-gray: #6B6E82; --color-gray-light: #C0C4D8; --color-off-white: #F6F7FB; /* Tipografía */ --font-display: 'Syne', sans-serif; --font-body: 'DM Sans', sans-serif; /* Bordes */ --radius-sm: 8px; --radius-md: 12px; --radius-lg: 20px; --radius-pill: 100px; /* Animaciones */ --ease-default: 0.3s cubic-bezier(0.4, 0, 0.2, 1); --ease-spring: 0.4s cubic-bezier(0.34, 1.56, 0.64, 1);
